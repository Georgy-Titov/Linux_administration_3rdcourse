## Описание

Данная лабораторная работа посвящена построению корпоративной инфраструктуры доступа к большим языковым моделям (LLM) в изолированном сетевом контуре предприятия. В рамках задания реализуется веб-прокси-платформа, объединяющая локальную LLM (LM Studio) и облачный сервис Сбер GigaChat, с централизованным управлением API-токенами через systemd и nginx.

Проект моделирует работу сетевого администратора в закрытом корпоративном сегменте, где доступ к внешним API строго контролируется, а взаимодействие с LLM осуществляется через защищённый reverse-proxy.

## Сценарий

Сетевой администратор отдела авиакомпании отвечает за внедрение интеллектуальной системы планирования рейсов на основе LLM.  

Инфраструктура предприятия:
- полностью изолированный корпоративный сегмент
- доступ во внешнюю сеть ограничен whitelist-политиками
- требуется централизованный доступ к LLM через веб-интерфейс

Для решения задачи разворачивается:
- локальный LLM сервер (LM Studio)
- облачный API GigaChat
- единый веб-портал с reverse proxy
- автоматическое обновление OAuth токенов
